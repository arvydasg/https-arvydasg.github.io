*** How to generate CodeAcademy and arvydasg.github.io blog
[2023-06-13 Tue]

Codeacademy - create a folder, create an org file, put content in
there. Add html_head(look previous files), make sure emacs config has
"Org-export-html" specified and evaluated for org-html-preamble and
org-html-postamble. Export to html with c+e hh, check if html has
styles, header, GTM tag and scroll to the top button. If yes - push to
git, done. Github will be listening to this repo and host the content
for you on /codeacademy. Images do not work when applied lightbox css
and lightbox js script because org exports images not with figure tag
but with div class figure. Stuff messes around then. No time now to
figure it out, but keep it in mind. Should be possible to modify
lightbox.js to look for other tags.

Okay, found a way how to make images work. Simply need to define image
with html source block in org file like such:



and then have lightbox.js and css defined in head tag. Watch
codeacademy/test/test.org for test sample. More about this is chat
with chatgpt here -
https://chat.openai.com/share/e1597a53-cf94-4672-b214-6c507f5b2089

arvydasg.github.io - is a main git repo that hosts websites. IF there
is a repo of the same person such as codeacademy, it will make it so
that it's content(Codeacademy) would appear like so -
arvydasg.github.io/ ~codeacademy~. Thats very nice.

To generate arvydas.github.io content, we must have
~arvydas.github.io_blog_content~ repo and define the location of this
folder in our org-static-blog emacs config.

Whenever we run ~org-static-blog-publish~ from now on - it will REBUILD
EVERYTHING that this directory contains and will export the files to
here /arvydasg.github.io/ folder(github repository cloned on your
computer). You then have to go to arvydasg.github.io folder, push the
changes to github and it will deploy for you to arvydasg.github.io and
since I have bought a domain name, to - arvydas.dev also.
